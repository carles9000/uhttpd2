Consideraciones para el editor workshop builder

Como principal dependencia (formio) sólo funciona para bootstrap
Conectar con la api de uhttpd2

Controles tipo input
  • Al editar el control, ir al Tab Layout y añadir atributos id y data-live
  • data-live tiene que contener un valor, por ejemplo, true de lo contrario no se crea
  • Dentro del Tab Validation incluiremos la validación a ejecutar en frontend
  • Dicha validación, no tiene nada que ver con backend, ya que tiene su propia lógica de validación
  • En el Tab display podemos personalizar los mensajes de validación, en el desplegable Custom Validation
  • Está indicado como proceder, por ejemplo con el siguiente formato :
    { "required": "Se requiere un ID para {{ field }}"}

Controles tipo button
  • Al editar el control, ir al Tab Layout y añadir atributos id, data-live y data-onclick
  • En data-onclick se vinculará la funcion que llama a la api que hemos definido en prg
  • En el Tab display dejar el campo action en blanco, de lo contrario nos fallará el enlace a nuestra api
  • También en el Tab display se define el tema, tamaño, icono, etc. del botón
